# 记忆规则

**最后更新**: 2026-03-04

> 本项目运行在 Claude Code 之上。Claude Code 有两层原生记忆（CLAUDE.md + Auto Memory），
> AI **必须**在工作过程中自动维护它们，不等用户手动要求。

---

## 1. 两层记忆系统

| | CLAUDE.md + .claude/rules/ | Auto Memory |
|---|---|---|
| **谁写** | 人类 + AI（本规则授权） | AI |
| **内容** | 项目描述、行为约束、入口索引 | 调试经验、偏好学习 |
| **持久化** | git-tracked（跨机器） | machine-local（可丢失） |
| **加载** | 每次会话完整加载 | 前 200 行 |

---

## 2. 自动触发规则（**MUST**，不可跳过）

AI 在以下事件发生时，**必须主动**检查并更新对应文件，不等用户要求。

### 2.1 CLAUDE.md 自动更新触发

CLAUDE.md 是项目入口索引。当项目结构发生变化时，AI 必须检查 CLAUDE.md 是否需要同步。

**通用原则**：CLAUDE.md 中引用的事实（命令、文件名、目录结构、skill 列表）发生变化时，同步更新对应章节。

- **数据型变更**（文件名、目录、计数等）→ AI 直接更新
- **语义型变更**（项目描述、工作流）→ AI 提议，用户确认后写入

**约束**：CLAUDE.md ≤ 200 行，是入口索引，不展开详细内容。

### 2.2 .claude/rules/ 自动更新触发

| 事件 | 动作 |
|------|------|
| 发现可复用的行为约束（"AI 在本项目必须..."） | **新建**或**追加到**对应 rules 文件 |
| 已有规则失效（流程废除、工具弃用、文件重命名） | **修改**或**删除**对应规则 |
| 规则冲突（两个文件说法矛盾） | 合并，保留一处完整定义 |
| 文章交付 / skill 完成 | 检查流程中产生的行为约束是否已沉淀到 rules |
| 用户说"记住这个规则" | 按 §3 决策树判断归属后写入 |

**约束**：自动触发只写项目级（仓库内）。User 级（`~/.claude/`）仅在用户明确要求时写入（见 §3 Q3）。

### 2.3 Auto Memory 写入指引

Auto Memory 是 Claude Code 的平台行为，AI 自动积累经验。本节定义**主动持久化时的优先级判断**：

1. **先判断**：这条经验是否足够稳定且项目通用？
   - 是 → 应写 CLAUDE.md 或 .claude/rules/（git-tracked，跨机器）
   - 否（临时/个人/不确定）→ Auto Memory（≤ 10 行）
2. **典型 Auto Memory 内容**：调试经验、尚未确认的偏好、临时工作断点

### 2.4 记忆升级（Auto Memory → git-tracked）

AI 在 Auto Memory 中发现与已有记忆重复或高频使用的经验时，主动**建议**升级（用户确认后执行）：

```
Auto Memory 中的高频经验
  ↓
项目级（所有开发者都需要）→ 建议写入 .claude/rules/ 或 CLAUDE.md
个人级（只有我需要）→ 建议写入 ~/.claude/rules/
```

### 2.5 Rules 变更后验证（**MUST**）

Rules 文件被修改后，AI **必须**执行验证，不等用户要求。

**触发场景**：重命名、拆分、合并、章节移动、交叉引用变更。

**验证三步**：

1. **扫描引用残留** — 旧文件名/旧章节号是否还被其他文件引用（`.claude/rules/*.md`、CLAUDE.md）
2. **验证指针可寻址** — 所有 `详见 xxx.md §N` 格式的指针，目标文件和章节存在
3. **跑项目测试** — 如项目有测试命令则执行

**历史文件豁免**：已归档内容不更新。

### 2.6 Rules 信息密度管控（**MUST**）

Rules 文件每次会话完整加载，直接消耗上下文窗口。AI 新增或修改 rules 内容时，**必须**检查密度预算。

**预算基线**：

| 指标 | 预算 |
|------|------|
| 始终加载总行数 | ≤ 800 行（rules + CLAUDE.md） |
| 全量总行数 | ≤ 1,800 行 |

**触发检查**：新增 rules 内容后，统计总行数是否超预算。超出 → 先压缩再合入。

**压缩策略**：

| 保留 | 删除或压缩 |
|------|-----------|
| 行为规则和约束指令 | 冗余"为什么"解释 |
| 决策树和判断流程 | 过长示例 |
| 表格型速查 | 重复的上下文说明 |

**写作风格**：规则用祈使句（"AI 必须..."），不写教程式说明。表格优于散文。指针优于复制。

## 3. 记忆归属决策树

AI 产出一条知识并考虑持久化时，**必须**按以下顺序判断：

```
输入：一条知识内容
  │
  ├─ Q1: 是行为约束（"AI 必须/不能做某事"）？
  │      是 → .claude/rules/
  │      否 ↓
  │
  ├─ Q2: 是项目描述（"这个项目是什么/怎么用"）？
  │      是 → CLAUDE.md
  │      否 ↓
  │
  ├─ Q3: 是个人偏好（非项目级）？
  │      是 → ~/.claude/rules/（用户确认后写入）
  │      否 ↓
  │
  ├─ Q4: 是临时 AI 工作上下文？
  │      是 → Auto Memory（≤ 10 行）
  │      否 ↓
  │
  └─ 不属于记忆系统 → 写入项目目录或 notes/
```

**速查表**：

| 内容类型 | 归属 |
|----------|------|
| "AI 必须自动提交" | .claude/rules/（行为约束） |
| "这个项目用于文章写作" | CLAUDE.md（项目描述） |
| "我喜欢 vim 键绑定" | ~/.claude/rules/（个人偏好） |
| "当前在写 LLM API 文章" | Auto Memory（临时上下文） |

## 4. 多人协作与环境可达性

### 4.1 协作规则

Git-tracked 文件（CLAUDE.md、`.claude/rules/`）是**共享记忆**——git push 后所有环境 pull 即可获得。

| 事件 | AI 必须做 |
|------|----------|
| 写入 `.claude/rules/` 或 CLAUDE.md | 提醒用户提交到 git |
| 写入 `~/.claude/rules/` | 提醒："此规则仅本机生效" |
| 修改 rules 文件后 | 提醒：需**新会话**才加载更新的 rules |

### 4.2 环境可达性

| 层级 | 文件 | 共享范围 | 丢失后果 |
|------|------|---------|---------|
| 团队共享 | CLAUDE.md、.claude/rules/ | git clone 即得 | 不会丢失 |
| 个人维护 | ~/.claude/rules/ | 仅本机 | 可恢复 |
| 环境私有 | Auto Memory、CLAUDE.local.md | 仅当前环境 | 可丢失（by design） |

**CLAUDE.local.md**：项目级本地覆盖（git-ignored），用于本机特有配置。优先级高于 CLAUDE.md。
